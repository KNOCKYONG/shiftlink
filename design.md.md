# ShiftLink 웹 디자인 기획 (Best Practice)

## 1. 디자인 철학
- **심플 & 직관적**: 근무자는 IT 전문가가 아님 → 학습 곡선 최소화
- **문제 해결 중심**: 불필요한 버튼/정보는 배제, 꼭 필요한 액션만 노출
- **시각적 가시성**: 피로도/편중/공백 같은 중요한 지표는 숫자+컬러로 즉시 인식
- **반응형 설계**: PC/태블릿/모바일 최적화
- **소통 강조**: 공유·알림·교환 요청 플로우는 클릭 수 최소화

---

## 2. 주요 화면 (IA/플로우)

### (A) 로그인/온보딩
- 이메일 OTP / Google / Microsoft 로그인
- 첫 진입 시 튜토리얼 (3슬라이드: 근무표 생성 → 공유 → 교환)

### (B) 대시보드
- 상단 요약 위젯 (KPI 카드)
  - 오늘 근무 인원 / 공백 인원
  - 연속 야간자 수
  - 초과근로 예상 경고
- 중앙 스케줄 뷰 탭 전환
  - **표 보기** (직원 × 날짜, 색상코드)
  - **캘린더 보기** (월간/주간 뷰, 드래그&드롭 수정)
  - **반영사항 탭** (승인/변경/경고 로그)
- 우측 패널
  - 교환/휴가 요청 리스트 (승인/거절 원클릭)
  - 알림 센터

### (C) 교대 교환(Trade) UX
- 직원이 셀 클릭 → “교환 요청”
- 교환할 대상 선택 (자동 제안 포함)
- 상대방 수락 → 관리자 승인 → UI 즉시 업데이트
- 진행 상태(요청중/수락/승인/거절)는 뱃지 컬러로 표시

### (D) 근무자 개인 뷰
- 본인 근무표 캘린더 뷰
- 누적 근무시간, 피로도 지표
- 공유 버튼 → 이메일/카카오톡/캘린더 연동

### (E) 관리자 설정 화면
- 제약 옵션 ON/OFF 토글 UI
- 제약 값 슬라이더/숫자 입력 (예: 연속 야간 2~5회)
- 규칙 저장 시 “시뮬레이션” 버튼 → 테스트 근무표 생성

## 3. 디자인 시스템

### 컬러 팔레트
- Primary (브랜드): 블루 (#2D7FF9) → 신뢰·소통
- Secondary: 민트 (#30CFCF) → 활기·균형
- Alert/Warning: 옐로 (#FFD43B), 레드 (#F65C5C)
- Neutral: 다크 그레이 (#2E2E2E), 라이트 그레이 (#F4F5F7)

### 타이포그래피
- 헤드라인: Inter Bold 24–32px
- 본문: Inter Regular 14–16px
- 숫자 지표: Monospace 폰트

### 컴포넌트
- 카드형 KPI 위젯
- 토글/슬라이더 기반 옵션 UI
- 탭 전환 (표/캘린더/반영사항)
- 드래그&드롭 캘린더 이벤트
- 알림 뱃지 (요청/경고/승인 상태 색상 구분)

---

## 4. UX Best Practice 포인트
- **1클릭 공유**: 이메일/카카오/캘린더 공유 버튼 제공
- **Undo 기능**: 잘못 편집 시 되돌리기
- **상태 가시성**: 교환/승인 등은 색상 뱃지 + 툴팁 제공
- **모바일 최적화**: 캘린더 뷰는 월간 → 주간 자동 축소
- **접근성 준수**: 색약 대비 팔레트, 키보드 내비게이션, 스크린리더 호환

---

## 요약
ShiftLink UI/UX 기획은:

- **대시보드형 요약 + 이중 뷰(표/캘린더) + 반영사항 로그**

- **간단한 교환 프로세스**와 **원클릭 공유**

- **제약 옵션화 + 시뮬레이션 기능**

을 중심으로 한 깔끔하고 직관적인 디자인입니다.
